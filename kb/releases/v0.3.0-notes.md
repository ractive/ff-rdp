# ff-rdp v0.3.0

v0.3.0 catches the released binary up with ~525 commits of work merged since
v0.2.0 (2026-05-06) — everything from roughly iter-66 onward. The headline is
**Firefox 152 compatibility**: two commands (`index` and `console`) were broken
in the v0.2.0 binary against current-stable Firefox and are fixed here.

> then this draft is published.

## Highlights

### Firefox 152 compatibility

- **`index`** — Firefox 152 changed how computed colors serialize (keywords
  like `red` where older Firefox returned `rgb(...)`), which broke page-map
  resolution. Fixed and normalized in
  [[iteration-114-live-suite-debt-zero]] (Theme A — Firefox-152 color drift).
- **`console`** — a fresh connection's console cache is now primed by calling
  `WebConsoleActor::start_listeners` before `get_cached_messages`, so the first
  `console` call after connect returns buffered messages instead of an empty
  list. See [[iteration-116-console-cache-start-listeners]].

### Cascade fidelity

- Cascade entries now carry `rule_actor_id`, letting consumers distinguish
  same-line inline stylesheets that were previously collapsed together.
  See [[iteration-115-cascade-rule-actor-id]].

### Daemon hardening & live-launch reliability

- Live-launch hardening (timeouts, race-free harness) landed in
  [[iteration-113-live-launch-hardening]], on top of the daemon-lifecycle and
  scalability work from the iter-90s/iter-100s series.

### Live-suite trust restoration

- The live end-to-end suite was brought to zero known debt in
  [[iteration-114-live-suite-debt-zero]]: Firefox-152 color drift normalized,
  self-launching legacy tests, and a self-hosted fixture server so live tests
  no longer depend on flaky external sites.

## CI / release-process changes (iter-117)

- The GitHub Actions live-test lane no longer runs per-PR. It was permanently
  red from environmental runner failures with no real signal; the authoritative
  live gate is local and enforced per-iteration via `check-iteration-ready`. The
  lane now runs on `workflow_dispatch`, at `release: published`, and as a weekly
  drift-canary `schedule` cron.

## Upgrade notes

- No breaking CLI changes. `--version` continues to report the semver plus the
  build git-sha and date (e.g. `ff-rdp 0.3.0 (<sha> <date>)`).
- Minimum supported Rust version stays 1.95.
