## Acceptance Criteria [0/N]

**Convention (mandatory):** Every AC checkbox MUST name the test that verifies it and state
the asserted post-condition. Format:

```
- [ ] <test_function_name>: <asserted post-condition>
```

Examples:
- `[ ] live_screenshot_full_page: PNG height ≥ scrollHeight × DPR`
- `[ ] mock_server_inject_watcher_resource: three resource types received with correct envelope`
- `[ ] live_network_headers: meta.source == "watcher" AND Content-Type present in headers.response`

An AC that lacks a named test is **not done**, even if the feature code is merged.
