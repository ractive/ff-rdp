# K0 — Unknown symptom (broad sweep fallback)

## Symptoms

- The user cannot characterize the problem.
- The router matched no other playbook.
- "Something's wrong with the page."

## Failing layer

Unknown — this playbook **does not diagnose**. It collects three
broad-spectrum captures and offers them back to the user as conversation
starters, then asks for a more specific symptom.

## Probe sequence

This is a flat collection sequence, not a tree. Run all four steps,
emit summaries, then prompt.

### 1. Console errors

```bash
ff-rdp console --level error --limit 10 --jq '.results'
```

### 2. Failed network requests

```bash
ff-rdp network --status '4..|5..' --detail --limit 10 \
  --jq '.results[] | {url, status, method}'
```

### 3. Interactive snapshot (orient the agent)

```bash
ff-rdp snapshot --interactive-only --limit 30
```

### 4. URL + tab state

```bash
ff-rdp eval 'location.href'
ff-rdp tabs --jq '.results'
```

## Output (always inconclusive)

The skill emits the standard "inconclusive" footer instead of a
diagnosis:

```
## Inconclusive — broad sweep complete

Captured:
- N console errors (top 3: …)
- M failed requests (top 3: …)
- K interactive elements visible

Three follow-up questions (answer any one):
1. Did anything trigger this (reload, click, route change), or was it
   broken from first page load?
2. Is there one specific element or action that fails, or does the
   whole page feel wrong?
3. Does it happen in another browser or only this one?
```

If the captures hint at a known playbook layer (e.g. a `ChunkLoadError`
in console, a 401 + `Set-Cookie`-less response in network), the skill
prepends a single hint line:

```
## Hint: this looks like an E1-flavored bug (ChunkLoadError in console)
```

…but does **not** auto-run the matched playbook. The user must
confirm.

## Red herrings

- One unrelated console warning is normal noise; don't chase a single
  message.
- Interactive snapshot showing many elements doesn't tell you anything
  diagnostic on its own — it's just orientation.

Fixture: `evals/fixtures/K0/`. A page with one console error

Demonstrates the broad-sweep capture + inconclusive footer.
